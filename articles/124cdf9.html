sql语句

发表于 2021-03-20 更新于 2021-03-26
我已开始练习

SQL语句学习：

  • 建库：

①创建数据库：create database db_name;

②删除数据库：drop database db_name;

③显示创建数据库的语句：show create database db_name;

④显示数据库语句：show databases;

⑤在创建数据库或者数据表的时候，是可以指定字符集的；

只需要在后面加上character set <字符集>；

⑥切换数据库 use db_name;

⑦查看当前使用的数据库 select database()；

  • 建表

mysql中的建表语句：

create table table_name（

属性字段 属性对应的类型以及大小

id varchar(8),

……

主键声明

）字符集等说明；

  • 修改表结构

①修改表名称：alter table 表名 rename 新表名;

②添加字段： alter table 表名 add 字段名称 字段定义;

③删除字段 ：alter table 表名 drop 字段名称；

④修改字段定义：alter table 表名 modify 字段名称 字段定义；

⑤修改字段名称以及字段定义：alter table 表名 change 旧字段名称 新字段名称 字段定义

注：字段定义包含字段类型，字段约束等

  • 查看表结构 ：desc table_name;

  • 增加数据：insert into 表名（字段1，字段2，……）values (value1,value2,……)；

    注：对于char和varchar 我们要对插入的值加双引号。

  • 查询数据：

    • 查询所有数据：select * from 表名称；

      注意：如果select * from 表1，表2；那么查询出来会出现两张表的全连接情况，做笛卡尔积运算；

    • 限制查询几条数据用limit来限制 ：

      • select * from table_name limit 3,4; 这句话表示查询输出表中的4,5,6,7,行的数据，即从第三行（不包括）开始，输出4行数据；
      • select * from table_name limit 5 offset 4;这句话表示查询输出表中5,6,7,8,9，行的数据，即从第四行（不包括)开始，输出5行数据；

    • 只查看某几个字段：select 字段1，字段2，……from 表名；

    • 还可以和distinct组合，去重查询，假如姓名列中有人名重复，那么我们只想统计人数，我们可以这样

      select distinct name from user；

    • where子句（用于过滤筛选），当使用where子句按条件查找的时候，我们通常要结合一些运算符：

      • 比较运算符

        大于，小于，等于，不等于；

        between……and……，用以显示在某一区间的值,包括两端的值；

        in ,用以显示in列表中的值，例如in(100,200);

        like 模糊查询，一般匹配_或者%；

        is null：判断是否为空；

      • 逻辑运算符

        and 组合条件需要同时满足；

        or 多个条件任一成立

        not 非，例如where not（age>10）;

  • 更新数据：

    • 更改某行的某列

      update 表名称 set 列名称=新值 where 某行的定义；这里某行的定义是为了约束到某一行，因为你不约束的话 ，这条update语句就会影响这列的所有行，当然这种情况一般是比较少的，所以一般我们都会加where子句来约束一下：比如有一张用户表，我要更改第四行的姓名：

      那么就用 update user set name=“帅哥” where id=4;

    • 更改某行的某几列

      update 表名称 set 列名称1=新值 ，列名称2=新值 where 某行的定义；

  • 删除数据（delete）

    • delete from table_name where 要删除的属性值，比如我要删除第四行的姓名

    那么我就用delete from user where name=“帅哥”；

    可以对字段起别名，用as来起别名 as关键字也可以省略

    sql内置函数

    sql里的这些函数一般和select和where等关键字组合使用

    聚合函数（aggregate function），这些聚合函数是可以写在一起的，比如select sum（列名），avg（列名） from tb_name;

    ①count（），一般返回匹配指定条件的行数。

    ​ 一般count（*）和count（1）执行效果一样，但是执行效率不同，而且执行逻辑不一样，虽然他俩都是用来统计表中所有字段的数目，但是count（1）可以统计到null值,另外一个不行，而且统计大量数据的时候，前者慢一点，也就是count（1）快一点

    • COUNT(*)
    • COUNT(1)
    • COUNT(column name)会忽略列值为null的情况
    • COUNT(DISTINCT column name)

    执行效率上，统计主键所在列肯定比较快，但是如果只有一个字段，count（*)最快，因为count（星号）统计的时候，要检索所有列，如果只有只一个字段，那就只有一列，这样省去检索列名的时间，所以直接统计行数，效率很高；

    ②sum（），一般返回某列的和

    ③max(),min():一般返回某列的最大或者最小值；

    ④concat():一般可以拼接多个表达式，并返回最后结果，例如select concat(avg(id),sum(id)) from deerli;

    ​ 甚至我们在进行更改数据表时也可以用这个函数，比如我们想批量给某列后面加上字符串”shuai”,

    ​ 这时候sql语句怎么写呢，就是update table_name set 列名=concat(列名，”shuai“);这是一个示例，sql语句里面很多都可组合使用的。

    ⑤concat_Ws()：一般可以在拼接过程中加上连接符，字符串连接符函数，注意W是大写的；而且一般是把连接符写到第一个参数的位置，后面写连接符要连接的字段；

    日期函数（date function）

    ①current_timestamp();返回数据库日期

    ②日期转换函数

    ③还有日期相减函数，用于计算日期之间的差

    ④还有时间和时间戳相互转换函数

    数值计算函数

    ①round（）：取小数点后几位

    ②floor（）:向下取整

    ③rand（）：取随机数

    小结：sql中有许多函数，以上是比较常见的

    Having语句

    ​ having语句主要是可以继续作过滤，功能比where子句要多一些，可以多层过滤，而且where子句没办法对结果再次查询。

    判断语句

    其中一种语法是if(判断条件，条件成立的结果，条件不成立的结果)；和其它关键字组合使用；

    另外一种常用的判断是case when 条件成立 then 返回值 else 返回值 end；

    排序语句

    order by语句，用于对表中数据进行降序排列：select * from table_name order by 列1，列2……；

    group by语句，分组，即可以单独计算分组中的数据，比如计算消费表里每人平均花费多少钱；

    ​ select name,avg(cost) from table_name group by name;

    ​ 这个语句就是按照名字来分组计算，最后就可以得到每人平均所花费的钱数；

    ​ 这个分组函数完成的就是相同数据进入同一组。
